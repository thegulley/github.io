Step-by-step guide

Setting up a
public GitHub
Pages site

How to host Claude artifacts as live, shareable web pages — for free.

Setup steps
1
Create a GitHub account

Go to github.com and click Sign up. Choose your username carefully — it becomes part of your permanent site URL forever.

https://yourusername.github.io
Tip Pick something you're comfortable sharing publicly — your name, a handle, or a project name.
2
Create your repository

A repository is just a folder for your project. This one has a special magic name that activates your website.

Click the + icon in the top-right → New repository
Name it exactly: yourusername.github.io using your actual username
Set visibility to Public
Click Create repository
Critical The repository name must match your username exactly, or the public URL won't work. E.g. if your username is thegulley, the repo must be named thegulley.github.io — not just github.io.
3
Create your first file

The repo starts empty. You need to add a file to create the main branch, which GitHub Pages requires before it will let you enable the feature.

On the repository page, look for the blue "Quick setup" box. Click the creating a new file link inside it.

Name the file index.html
Paste in: <h1>Coming soon</h1>
Click the green Commit changes button top-right
Click Commit changes again in the popup to confirm
Why this matters Without this step, the Branch dropdown in Settings will be empty and you won't be able to enable GitHub Pages.
4
Enable GitHub Pages
Click Settings in the top tab bar of your repo
Click Pages in the left sidebar
Under "Source" select Deploy from a branch
Under "Branch" select main, leave folder as / (root)
Click Save
Success indicator You'll see a green banner: "GitHub Pages source saved." Your site is now being built.
5
Upload your artifacts

Go back to your repo's main page. From now on, every Claude artifact you want to share follows the same simple process:

Click Add file → Upload files
Drag in your .html file saved from Claude
Write a short commit message e.g. add calculator artifact
Click Commit changes

Your file is now live at:

https://yourusername.github.io/yourfilename.html
Good to know
Tips & troubleshooting
```
Changes are slow
After every upload, wait 1–3 minutes before checking the URL. GitHub needs time to rebuild the site.
Page looks outdated
Do a hard refresh: Ctrl+Shift+R on Windows or Cmd+Shift+R on Mac. Or open the URL in a private/incognito window.
Updating a file
Upload a new file with the exact same filename. It overwrites the old version automatically. Wait 1–3 minutes, then hard refresh.
Wrong repo name
If you named the repo incorrectly, go to Settings → General and rename it to yourusername.github.io. The URL won't work otherwise.
Saving Claude artifacts
In Claude, use the download button on the artifact, or copy the code → paste into Notepad (Windows) or TextEdit (Mac) → save as name.html. Make sure it saves as .html not .txt.
React artifacts
React (.jsx) files don't work as standalone pages. Ask Claude: "Give me this as a single self-contained HTML file I can host on GitHub Pages."
```
